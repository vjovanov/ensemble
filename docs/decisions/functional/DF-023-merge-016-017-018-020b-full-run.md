## 3. jq-3238 root cause — NOT DF-017; it is a case-set subset-fix (DF-020b stop-clause)

Hypothesis going in was §DF-017 `filterGraphNoise` hiding jq's generated `parser.c`. **Disproven:** across all
three seeds, base (3/3) and merge (1/3) **both edit the same file `src/builtin.c`** — not a generated file,
and `builtin.c` matches none of `filterGraphNoise`'s noise patterns. So the edit site is correct; the failure
is **fix content/completeness**, which is exactly jq-3238's documented class (§DF-010-explore-surface-test-caseset:
"the fix covers the case seen, not the case-set graded"). The most consistent cause is **§DF-020b's "once you
can answer, stop searching" clause** cutting case-set exploration short during a bug fix — the same mechanism
that softens simdjson-2178 (content-not-location, §DF-020b.3). It is not a misdirection or a filter artifact.

## 4. Verdict & recommendation

- **Adopt the cost/token win and the §DF-016 correctness gain.** −34% cost / −43% tokens vs classic at +1
  pass@3 is a real improvement; svelte-15115 0→3/3 is a clean, attributable win.
- **Do NOT freeze this exact bundle as the next base until jq-3238 is recovered.** A 3/3→1/3 regression on a
  previously-solid instance is above the noise floor and clearly attributable to a fix-completeness drop.
- **Ablation DONE — §DF-020b confirmed as the cause; fix identified.** The merge *minus* DF-020b
  (`ablate-no020b-k3`, K=3, keep 016/017/018) **recovered both regressors**: jq-3238 1/3→**2/3**, simdjson
  1/3→**2/3** (back to base). So 016/017/018 are clean and DF-020b's "stop searching" clause is the sole
  cause. Cost confirms the mechanism (with DF-020b simdjson was cheaper $0.51 but wrong; without it $0.97 and
  right — the stop-clause buys cheapness by under-exploring the case-set).
- **Fix & re-merge plan:** gate DF-020b's decisive/stop-clause **off failing-test / bug-fix tasks** (let
  §DF-008 root-cause tracing govern those), keeping it for pure understanding/flow questions where it won
  (grpc −46%, etc.). The re-merged bundle should then keep the −12%/−34% cost win + svelte-15115 0→3/3
  **and** restore jq-3238 (~2-3/3) and simdjson (~2/3) — net correctness moving from +1 to roughly +3, a
  clean adopt. Re-run the full base002-30 K=3 after the gate fix to confirm before freezing the next base.
- Controls held: the other 22 instances are flat-to-better; the cost win is broad (grpc −46%, bat −42%, vue
  −20 to −32%, dayjs-2399 −21%), not carried by the regressors.
